title: Connector Reference

Connectors are the integrations the engine uses to query external intelligence sources at query time. Each connector targets a specific data category and requires outbound HTTPS access to one or more external hostnames.

Use `GET /v1/connectors` to see the current status of all connectors in your deployment.

## Credential modes

Each connector operates in one of two credential modes:

| Mode | Description |
|------|-------------|
| `intrace_managed` | Intrace provisions and maintains the credentials for this connector. No action required from your organization. |
| `byok` | Bring your own key. Your organization supplies credentials for this connector, typically because you have an existing contract with the data provider. |

BYOK connectors require connector-specific environment variables. These are documented per connector below.

## Connector categories

### Sanctions & watchlists

These connectors check identifiers against official sanctions lists and law enforcement watchlists. They are always queried regardless of `depth`.

| Connector ID | Source | Credential mode | Required egress |
|---|---|---|---|
| `sanctions_ofac` | OFAC SDN and consolidated lists | Intrace-managed | `api.treasury.gov:443` |
| `sanctions_un` | UN Consolidated Sanctions List | Intrace-managed | `api.un.org:443` |
| `sanctions_eu` | EU Restrictive Measures | Intrace-managed | `api.sanctionsmap.eu:443` |
| `sanctions_uk` | UK OFSI Financial Sanctions | Intrace-managed | `ofsistorage.blob.core.windows.net:443` |

### Commercial data

These connectors query licensed commercial identity and risk data providers. Coverage is primarily US-based with international options available depending on your license.

| Connector ID | Source | Credential mode | Required egress |
|---|---|---|---|
| `commercial_primary` | Intrace-licensed commercial data | Intrace-managed | Provided in license documentation |
| `commercial_byok_lexisnexis` | LexisNexis Risk Solutions | BYOK | `api.lexisnexis.com:443` |
| `commercial_byok_acurint` | Acurint | BYOK | `api.acurint.com:443` |

#### BYOK configuration: LexisNexis

```bash
CONNECTOR_LEXISNEXIS_API_KEY=your_api_key
CONNECTOR_LEXISNEXIS_ACCOUNT_ID=your_account_id
```

#### BYOK configuration: Acurint

```bash
CONNECTOR_ACURINT_API_KEY=your_api_key
```

### Open source intelligence (OSINT)

These connectors gather publicly available data from open sources. Queried at both `standard` and `deep` depth.

| Connector ID | Coverage | Credential mode | Required egress |
|---|---|---|---|
| `osint_domain` | Domain registration, DNS, WHOIS | Intrace-managed | `rdap.org:443`, `whois.iana.org:443` |
| `osint_ip` | IP geolocation, ASN, hosting provider | Intrace-managed | `api.ipinfo.io:443` |
| `osint_breach` | Known data breach exposure | Intrace-managed | Provided in license documentation |

---

### Social & messaging platforms

These connectors query public profile data from social and messaging platforms. Some platforms restrict automated access — coverage depends on current platform API availability and your license.

| Connector ID | Platform | Credential mode | Required egress |
|---|---|---|---|
| `social_telegram` | Telegram public channels and profiles | Intrace-managed | Provided in license documentation |
| `social_platforms` | Major social platforms | Intrace-managed | Provided in license documentation |

Dark web connectors are available on select license tiers. Contact Intrace for availability.

## Verifying egress

After configuring your firewall, verify that all required connectors are reachable:

```bash
curl -H "X-API-Key: your_api_key" https://your-hostname/v1/connectors
```

Each connector should show `"status": "reachable"`. For any connector with `"status": "unreachable"`, check:

1. The firewall egress rule for the connector's required hostname and port
2. DNS resolution for the connector's hostname from the deployment host
3. The connector's credential configuration if in BYOK mode (invalid credentials return `"status": "unreachable"` with `"unavailability_reason": "credentials_invalid"`)

You can also check a single connector in real time:

```bash
curl -H "X-API-Key: your_api_key" https://your-hostname/v1/connectors/sanctions_ofac/health
```
